```bash
# ─────────────────────────────────────────────────

cd server-monitoring

# ─────────────────────────────────────────────────

# ─────────────────────────────────────────────────

# ─────────────────────────────────────────────────
nano config.py

# ─────────────────────────────────────────────────
#  6 — Launch 🚀

python run.py
```

## 🔄 Background Thread

```python
# run.py — daemon thread starts alongside the Flask app
tracking_thread = threading.Thread(target=collect_server_data, daemon=True)
tracking_thread.start()
```

<br/>

```
   ┌──────────────┐    every 30s     ┌───────────────┐
   │  Flask App   │ ───────────────► │  SSH into     │
   │  (running)   │                  │  each server  │
   └──────────────┘                  └──────┬────────┘
                                            │  run `who`
                                            ▼
                                     ┌─────────────────┐
                                     │  Parse sessions  │
                                     │  + login times   │
                                     └────────┬─────────┘
                                              │
                               ┌──────────────┴──────────────┐
                               ▼                             ▼
                       ┌──────────────┐           ┌──────────────────┐
                       │   Update     │           │  Mark as ended   │
                       │ tracking     │           │  if gone from    │
                       │   .json      │           │  `who` output    │
                       └──────────────┘           └──────────────────┘
```

<br/>

| Step | Action |
|:----:|--------|
| `①` | SSHs into each configured server every **30 seconds** |
| `②` | Runs `who` — captures sessions with **actual login times** |
| `③` | Parses output and writes to `user_tracking.json` |
| `④` | Sessions gone from `who` are automatically **marked as ended** |

## 👥 SSH User Tracking — How It Works

<div align="center">
<img src="https://readme-typing-svg.demolab.com?font=Fira+Code&size=15&duration=3000&pause=800&color=34D399&center=true&vCenter=true&width=720&lines=Parses+native+Linux+%60who%60+—+not+%60last%60%2C+not+%60w%60;Captures+actual+login+times%2C+never+poll+times;IP+%E2%86%92+Name+mapping+for+human-friendly+display;Auto-filters+all+noise+%E2%80%94+only+real+users+shown" alt="Tracking typing" />
</div>

<br/>

The tracker SSHs in and runs `who`, getting **real** login times — not when the poller checked:

```
$ who
─────────────────────────────────────────────────────────────────
sac       pts/1     2026-02-10 11:08  (192.168.3.208)
│         │         │                 │
│         │         │                 └── Client IP (user's machine)
│         │         └── Actual login time (NOT the poll time)
│         └── Terminal (pts = SSH session, tty = physical console)
└── Username on the remote server
─────────────────────────────────────────────────────────────────
```

Each session is stored in `user_tracking.json`:

```json
{
  "login_time":   "2026-02-10T11:08:00",   ← from `who`, never changes
  "first_seen":   "2026-02-10T11:08:00",   ← first poll that found the session
  "last_seen":    "2026-02-10T17:30:00",   ← most recent poll with session active
  "terminal":     "pts/1",
  "logout_time":  null                      ← set when session disappears from `who`
}
```

> ⏱️ &nbsp;**Session Duration** &nbsp;=&nbsp; `last_seen` &nbsp;−&nbsp; `first_seen`

<br/>

### 🚫 &nbsp;Auto-Filtered Session Types

| Type | Example | Why Filtered |
|------|---------|:------------:|
| Server → Server SSH | `192.168.2.13 → node` | Not an end user |
| Physical display sessions | `:0` &nbsp;·&nbsp; `:1` | Console login, not SSH |
| Tmux / Screen sub-sessions | `:pts/9:S.0` | Internal multiplexer |
